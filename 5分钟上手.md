# 5 分钟上手

本指南面向第一次接手本项目的同学，目标是用最短时间完成：

- 本地打开项目
- 云函数可用
- 页面能看到真实数据

## 0. 你需要准备

- 微信开发者工具（已登录）
- Node.js（建议 18+）
- 云开发环境可访问权限

## 1. 拉取并打开项目（约 1 分钟）

1. 拉代码到本地（已拉取可跳过）。
2. 打开微信开发者工具，选择项目根目录。
3. 确认已识别：
   - 小程序目录：miniprogram/
   - 云函数目录：cloudfunctions/

## 2. 安装云函数依赖（约 1 分钟）

在项目根目录执行（PowerShell）：

```powershell
Get-ChildItem cloudfunctions -Directory | ForEach-Object {
  if (Test-Path "$($_.FullName)\package.json") {
    Push-Location $_.FullName
    npm install
    Pop-Location
  }
}
```

## 3. 最小部署顺序（约 2 分钟）

先部署这 4 个云函数，页面就能跑通主链路：

1. getUserProfile
2. syncUserProfile
3. getRealtimeFx
4. riskAnalyzer

然后按需部署增强能力：

- aiPortfolioAdvisor（组合建议）
- alertMonitor（提醒）
- aiFxMonitor（汇率监测）
- getXishuAssets / syncXishuAssets（悉数云同步）
- aiParseAsset / aiEstimateResale（OCR 和估值）
- tushareImport（历史汇率导入）

## 4. 环境变量最小清单（按需）

不是全部必填，只有用到对应能力才需要：

- SILICONFLOW_API_KEY（OCR/残值估算）
- TUSHARE_TOKEN（历史汇率导入）
- ALERT_TEMPLATE_ID、ALERT_PAGE_PATH（订阅消息提醒）
- ICON_LIBRARY_JSON（自定义图标库，可选）

## 5. 30 秒验收清单

按顺序点这几项，确认项目已可用：

1. 首页可显示汇率和总资产（无报错）。
2. 新增一张银行卡后，重新进首页能看到数据。
3. 进入“云函数诊断”页，执行：
   - getUserProfile（成功）
   - syncUserProfile（成功）
   - riskAnalyzer（成功）
4. 进入“悉数”页，新建一条资产并返回后仍存在。

## 6. 常见问题（快速定位）

### 6.1 提示“AI函数未部署到当前云环境”

- 检查函数是否部署在当前 env。
- 检查调用环境和部署环境是否一致。

### 6.2 OCR/估值失败

- 检查 aiParseAsset 或 aiEstimateResale 是否部署。
- 检查 SILICONFLOW_API_KEY 是否配置。

### 6.3 风险分析没结果

- 检查 riskAnalyzer 是否部署。
- 检查首页是否已有银行卡数据。

### 6.4 提醒不触发

- 检查 alertRules 配置（code/direction/threshold）。
- 定时模式下检查 alertMonitor 的 trigger 是否生效。
