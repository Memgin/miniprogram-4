# 多币种资产与悉数管理小程序详细说明书

## 1. 项目概述

本项目是一个基于微信小程序云开发的综合资产管理工具，包含两大核心域：

- 财务资产域：多银行卡、多币种汇率、风险分析、压力测试、提醒规则、组合建议。
- 悉数资产域：生活物品记录、日均成本与回本、OCR 识别录入、AI 残值估算、云端图标库。

项目采用前后端一体化云开发架构：

- 前端：微信小程序原生（WXML/WXSS/JS）。
- 后端：云函数（Node.js）。
- 数据：云数据库集合 user_configs 与 rates_history。
- 可视化：ECharts。

---

## 2. 技术栈与工程结构

### 2.1 技术栈

- 小程序运行框架：微信小程序原生
- 云能力：wx.cloud + 云函数 + 云数据库
- 图表：ECharts（通过 ec-canvas 组件接入）
- AI/外部服务：
  - 汇率来源：open.er-api.com、frankfurter.app
  - OCR/估值模型接入：SiliconFlow（可配置模型）
  - 历史汇率导入：Tushare

### 2.2 目录结构（核心）

- miniprogram/: 小程序前端代码
- cloudfunctions/: 云函数代码
- ai/: 本地训练与推理脚本（PyTorch/ONNX）
- assets_archive/: 资源归档
- uploadCloudFunction.sh: 云函数部署脚本（当前为占位/待完善）

## 4. 全局数据模型与同步机制

## 4.1 globalData 关键字段

在 miniprogram/app.js 中维护全局状态，核心包括：

- bankCards: 银行卡与币种余额
- depositTarget / goalBuckets: 目标体系（兼容旧版单目标）
- selectedRateCodes: 首页展示汇率代码
- savedStressScenarios / latestStressResult: 压力测试
- alertRules / alertHistory / lastAlertCheckAt: 汇率提醒
- latestRiskSummary / latestAdvice: 风险分析与建议
- fxOrderPlans / cashflowPlans / maturityPlans: 计划器数据
- privacyMode / biometricEnabled: 隐私与生物认证开关
- exchangeRates: 实时汇率缓存

## 4.2 本地缓存

通过 wx.getStorageSync / wx.setStorageSync 存储，包括但不限于：

- bankCards
- depositTarget
- goalBuckets
- selectedRateCodes
- savedStressScenarios
- latestStressResult
- alertRules
- alertHistory
- lastAlertCheckAt
- latestRiskSummary
- latestAdvice
- fxOrderPlans
- cashflowPlans
- maturityPlans
- privacyMode
- biometricEnabled
- exchangeRates
- XISHU_ASSETS_V1（悉数本地数据）

## 4.3 云端同步策略

- 启动阶段：调用 getUserProfile 拉取云端用户配置。
- 写入阶段：app.sync() 触发全量同步，1.5 秒防抖后调用 syncUserProfile。
- 兜底机制：云拉取失败时使用本地缓存继续运行。

## 5. 页面说明（前端）

## 5.1 首页 pages/index/index

定位：资产总览与风控中枢。

主要能力：

- 拉取实时汇率并计算总资产（折合人民币）。
- 汇率展示列表（可配置展示币种）。
- 多目标进度管理（goalBuckets）。
- 风险分析（riskAnalyzer）与风险贡献图。
- 压力测试场景管理与结果持久化。
- AI 汇率监测（aiFxMonitor）。
- AI 组合建议（aiPortfolioAdvisor）。
- 汇率提醒规则创建、试跑与历史记录。
- 隐私模式与生物认证开关。

关键云函数调用：

- riskAnalyzer
- aiPortfolioAdvisor
- alertMonitor（manual=true）
- aiFxMonitor

## 5.2 悉数页 pages/xishu/xishu

定位：生活资产全生命周期管理。

主要能力：

- 资产/心愿双列表。
- 多维筛选与搜索：状态、分类、置顶、排序。
- 编辑器支持：价格币种换算、关联物品、备注图。
- 计费模式：按时间 / 按次打卡。
- OCR 导入：调用 aiParseAsset 识别图片信息。
- 残值预测：调用 aiEstimateResale。
- 图标选择：云图标库 + 本地图标 + 最近使用。
- 云同步：getXishuAssets / syncXishuAssets。

关键本地与云字段：

- STORAGE_KEY: XISHU_ASSETS_V1
- cloudSyncState / cloudSyncText: 同步状态提示

## 5.3 银行卡总览 pages/bankCards/bankCards

- 风险带展示（结合 riskAnalyzer 结果）。

## 7. 云函数总览

## 7.1 用户配置与同步

### getUserProfile

用途：读取当前 OPENID 对应用户配置。

入参：无。

返回：

- success: true/false
- data: 用户配置对象（新用户返回默认结构）

涉及集合：

- user_configs（docId = OPENID）

### syncUserProfile

用途：全量覆盖写入用户配置。

主要入参：

- bankCards, depositTarget, goalBuckets
- selectedRateCodes
- savedStressScenarios, latestStressResult
- alertRules, alertHistory, lastAlertCheckAt
- latestRiskSummary, latestAdvice
- fxOrderPlans, cashflowPlans, maturityPlans
- privacyMode, biometricEnabled

返回：

- success: true/false
- errMsg（失败时）

写入说明：

- selectedRateCodes 统一转小写。
- alertHistory 最多保留 20 条。
- updateTime 使用 db.serverDate()。

## 7.2 汇率数据

### getRealtimeFx

用途：聚合实时汇率。

数据源顺序：

1. open.er-api.com
2. api.frankfurter.app（失败回退）

返回：

- success
- rates（含 CNY 基准、大小写键、MYR/RM 映射、provider、updateTime）

## 7.3 风险与建议

### riskAnalyzer

用途：组合风险建模。

入参：

- bankCards
- currentRates

核心逻辑：

- 计算币种敞口与总资产。
- 优先从 rates_history 读取历史序列，不足时回退外部 API。
- 计算 VaR95、VaR99、年化波动率、风险贡献、组合序列。

返回字段：

- success
- totalValue
- var95 / var99
- volatility
- topExposures
- riskContributions
- portfolioSeries
- sampleSize
- confidenceHint

涉及集合：

- rates_history

### aiPortfolioAdvisor

用途：基于持仓、风险与压力测试生成规则化建议文案。

入参：

- bankCards
- currentRates
- stressResult
- riskSummary

返回字段：

- success
- generatedAt
- headline
- summary
- insights
- actions

说明：

- 不依赖外部大模型密钥。

## 7.4 提醒与监测

### alertMonitor

用途：汇率提醒规则检查，支持手动与定时巡检。

模式一：手动（manual=true）

- 入参：alertRules, currentRates
- 返回：triggers, checkedAt, historyEntry

模式二：定时巡检（manual=false）

- 遍历 user_configs 用户规则。
- 更新 alertHistory 与 lastAlertCheckAt。
- 可选发送订阅消息（需 openapi 权限和环境变量）。

环境变量：

- ALERT_TEMPLATE_ID
- ALERT_PAGE_PATH

定时器：

- config.json: 0 0 9-23 * * * *

### aiFxMonitor

用途：单币种短期监测（EWMA + 年化波动率 + 信号分类）。

入参：

- symbol（默认 USD）
- seq_len（默认 20）

返回字段：

- success
- method: ewma_js
- data_source: history_api / spot_api
- current, pred
- expected_change_pct
- volatility
- risk_level
- signal
- sample_size

## 7.5 悉数资产与图标

### getXishuAssets

用途：读取当前用户 xishuAssets。

返回：

- success
- data: xishuAssets 数组

涉及集合：

- user_configs.xishuAssets

### syncXishuAssets

用途：同步 xishuAssets。

入参：

- xishuAssets

返回：

- success
- errMsg（失败时）

说明：

- 包含 update / add / duplicate 兜底路径。
- 同时更新 xishuUpdateTime 与 updateTime。

### getIconLibrary

用途：返回可选图标列表。

数据来源：

- 内置 DEFAULT_ICON_LIBRARY
- 可被环境变量 ICON_LIBRARY_JSON 覆盖

返回：

- success
- data: 标准化图标数组（id, name, category, url）

## 7.6 AI 识别与估值

### aiParseAsset

用途：资产图片 OCR/语义抽取。

入参：

- imageBase64
- apiKey（可选，优先级低于环境变量）

环境变量：

- SILICONFLOW_API_KEY
- SILICONFLOW_API_URL
- SILICONFLOW_VISION_MODEL

返回：

- success
- data: { name, price, date, raw }
- error（失败时）

特点：

- 多模型候选、图像上传 URL 与 DataURL 双策略。
- 先 JSON 解析，失败后正则兜底抽取。

### aiEstimateResale

用途：二手残值预测。

入参：

- asset: { name, category, price, holdDays }
- apiKey（必填）

返回：

- success
- data: { estimatedPrice, depreciationRate, marketTrend, sourceText, modelText }

说明：

- 当前模型为 Qwen/Qwen2.5-7B-Instruct。

## 7.7 历史汇率导入

### tushareImport

用途：从 Tushare 导入历史汇率至 rates_history。

入参：

- api_name（默认 fx_daily）
- token（event.token 或环境变量 TUSHARE_TOKEN）
- start_date / end_date

返回：

- success
- inserted
- updated
- total
- start_date / end_date

定时器：

- config.json: 0 10 18 * * * *

涉及集合：

- rates_history（按 symbol + date 去重更新）

## 11. 安全与合规建议

1. 不建议在前端长期保留可用的模型密钥或 fallback key。
2. 建议将所有关键密钥统一迁移到云函数环境变量。
3. user_configs 建议设置为仅数据所有者可读写。
4. 对 OCR/估值接口返回做字段校验与限流，避免异常成本。
5. 建议为 rates_history 建立生命周期策略或归档策略，控制数据量。

## 12. 常见问题与排障

## 12.1 AI 监测提示函数未部署

现象：页面提示“AI函数未部署到当前云环境”。

处理：

- 确认 aiFxMonitor 已上传到当前 env。
- 检查调用环境与部署环境是否一致。

## 12.2 OCR 识别失败或超时

处理：

- 检查 aiParseAsset 是否部署。
- 检查 SILICONFLOW_API_KEY 是否已配置。
- 压缩图片后重试（函数内对大图有限制）。

## 12.3 汇率提醒不触发

处理：

- 手动模式下确认 alertRules 的 code、direction、threshold。
- 定时模式下检查 alertMonitor 的 trigger 是否已生效。
- 如要发订阅消息，确认 ALERT_TEMPLATE_ID 与权限。

## 12.4 历史风险样本不足

处理：

- 使用 tushareImport 补充 rates_history。
- riskAnalyzer 在样本不足时会回退外部接口，但稳定性低于本地历史库。
